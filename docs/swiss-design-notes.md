# Swiss Design Notes For This Template

This template follows six Swiss principles as implementation constraints.

## 1) Grid Systems

- Primary layout is a single readable column for reports.
- Optional 12-column utilities are available for structured subsections.
- Alignment decisions should follow the grid, not visual guesswork.

## 2) Typography

- Helvetica-first stack to keep tone neutral and legibility high.
- One family for text hierarchy; distinction comes from scale and weight.
- Type ratio is controlled by CSS tokens for consistency.

## 3) White Space

- Use spacing tokens only (`--s1` to `--s7`).
- Prefer clear separation between sections over dense blocks.
- Empty space is intentional and improves comprehension.

## 4) Objectivity

- Content is primary; decoration is secondary.
- One accent color is reserved for meaningful emphasis.
- Visual effects are limited to what supports communication.

## 5) Reduction

- Remove repeated ideas and redundant headings.
- Keep tables concise and evidence-oriented.
- If a block does not add insight, remove it.

## 6) Clarity

- Make hierarchy obvious through heading scale and spacing.
- Use short paragraphs and explicit section titles.
- Structure for fast scanning before deep reading.

## Suggested Editing Routine

1. Draft all content without style concerns.
2. Remove non-essential sections and filler text.
3. Enforce heading and spacing consistency.
4. Validate print preview before final export.
